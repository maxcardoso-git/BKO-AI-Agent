```typescript
// Pseudocode — adapt to actual method signature
function renderInjectedCorrections(corrections: Array<{aiText: string; humanText: string; diffDescription: string; similarity: number | null}>): string {
  if (!corrections || corrections.length === 0) return '';

  const lines = corrections.map((c, idx) => {
    const sim = c.similarity != null ? ` (similaridade ${(c.similarity * 100).toFixed(0)}%)` : '';
    return `### Exemplo ${idx + 1}${sim}
**Rascunho IA original:**
${c.aiText}

**Versao corrigida pelo operador:**
${c.humanText}

**Resumo da correcao:**
${c.diffDescription}
---`;
  }).join('\n\n');

  return `
## Exemplos de Correcoes Humanas Anteriores (Aprendizado)

Os exemplos abaixo mostram como operadores corrigiram rascunhos previos para casos similares. USE essas correcoes como guia: evite os padroes que foram corrigidos e adote o estilo das versoes humanas.

${lines}
`;
}
```

Then in the existing prompt assembly (wherever the draft prompt is concatenated), insert:
```typescript
const correctionsBlock = renderInjectedCorrections(
  (input.humanCorrections as Array<{aiText: string; humanText: string; diffDescription: string; similarity: number | null}>) ?? []
);
// Concatenate into the prompt body in the appropriate position
```

WHY before generation instructions, after operator note: prompt-engineering best practice — examples should be near the task description; LLM weighs more recent tokens heavier. Operator note is highest-priority context (current case), examples come next.

WHY truncated to 800 chars per text (done in Task 1): keeps prompt under token budget — 3 examples * 2 fields * 800 = 4.8K chars overhead.
