**memory-retrieval.service.ts:**
- `@Injectable()` class `MemoryRetrievalService`
- Constructor injects `DataSource` from `typeorm`
- `async findSimilarCases(embedding: number[], tipologyId: string, limit = 3)`: raw SQL query on `case_memory` table. Select columns: `id, "complaintId", "tipologyId", metadata, 1 - (embedding <=> $1::vector) AS similarity`. Filter `WHERE "tipologyId" = $2`. Order `embedding <=> $1::vector ASC`. Limit $3. Use `pgvector.toSql(embedding)` for $1. Return typed as `Array<{ id: string; complaintId: string; tipologyId: string; metadata: Record<string,unknown>; similarity: number }>`.
- `async findSimilarCorrections(embedding: number[], tipologyId: string, limit = 3)`: same pattern on `human_feedback_memory` table. Select: `id, "aiText", "humanText", "diffDescription", "correctionCategory", 1 - (embedding <=> $1::vector) AS similarity`. Same filter/order/limit pattern.
- Import `pgvector` from `pgvector` (not subpath). Use `import * as pgvector from 'pgvector'` — the established pattern from PersistMemory skill in Phase 5.

**memory-feedback.service.ts:**
- `@Injectable()` class `MemoryFeedbackService`
- Constructor injects `DataSource` and `ModelSelectorService` (for embedding generation)
- `async persistFeedback(aiText: string, humanText: string, diffDescription: string, complaintId: string, tipologyId: string | null): Promise<void>`:
  1. Generate embedding: `const { embedding } = await embed({ model: this.modelSelector.getEmbeddingModel(), value: humanText })` — embed the human-corrected text so future searches find corrections similar to the human's version
  2. Insert into `human_feedback_memory` using raw SQL: `INSERT INTO "human_feedback_memory" ("id","aiText","humanText","diffDescription","correctionCategory","correctionWeight","embedding","complaintId","tipologyId") VALUES (gen_random_uuid(),$1,$2,$3,$4,$5,$6::vector,$7,$8) RETURNING "id"` with params `[aiText, humanText, diffDescription, 'response_edit', 1.0, pgvector.toSql(embedding), complaintId, tipologyId ?? null]`
  3. If embed() or insert throws, log the error but do NOT re-throw — memory persistence is non-critical and should not break the review flow

- Add two optional fields to the `PromptContext` interface:
  ```typescript
  similarCases?: Array<{ metadata: Record<string, unknown>; similarity: number }>;
  humanCorrections?: Array<{ aiText: string; humanText: string; diffDescription: string; similarity: number }>;
  ```
- In `buildDraftResponsePrompt()`, after the existing persona/template/mandatory fields are appended to the system prompt array, add:
  ```
  if (ctx.similarCases && ctx.similarCases.length > 0) {
    system.push('', '## Casos similares resolvidos anteriormente:');
    for (const c of ctx.similarCases) {
      system.push(`- Similaridade: ${(c.similarity * 100).toFixed(0)}% | ${JSON.stringify(c.metadata)}`);
    }
  }
  if (ctx.humanCorrections && ctx.humanCorrections.length > 0) {
    system.push('', '## Correcoes humanas em respostas anteriores similares:');
    for (const h of ctx.humanCorrections) {
      system.push(`- Correcao (${(h.similarity * 100).toFixed(0)}% similar): ${h.diffDescription}`);
      system.push(`  Versao humana aprovada: ${h.humanText.slice(0, 300)}`);
    }
  }
  ```
