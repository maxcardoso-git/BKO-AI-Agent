    3. Initialize shadcn/ui:

       - Accept defaults: new-york style, zinc base color
       - This sets up cn util, tailwind config, components.json
       - Install needed components: `npx shadcn@latest add button input label card table badge select`

    4. Create `frontend/.env.example`:
       ```
       SESSION_SECRET=your-session-secret-at-least-32-chars-long
       BACKEND_URL=http://localhost:3001
       ```
       Also create `.env.local` with the same values for dev.

    5. Create `src/lib/session.ts`:
       - import 'server-only'
       - import { SignJWT, jwtVerify } from 'jose'
       - import { cookies } from 'next/headers'
       - const secretKey = process.env.SESSION_SECRET
       - const key = new TextEncoder().encode(secretKey)
       - encrypt(payload): new SignJWT(payload).setProtectedHeader({ alg: 'HS256' }).setIssuedAt().setExpirationTime('8h').sign(key)
       - decrypt(session): jwtVerify(session, key, { algorithms: ['HS256'] }), return payload. Catch errors and return null.
       - createSession(accessToken, user): encrypt { accessToken, userId: user.id, role: user.role, email: user.email, name: user.name }, set 'session' cookie with httpOnly: true, secure: process.env.NODE_ENV === 'production', sameSite: 'lax', path: '/', maxAge: 8 * 60 * 60
       - deleteSession(): delete 'session' cookie
       - IMPORTANT: `await cookies()` — cookies() is async in Next.js 15+

    6. Create `src/lib/dal.ts`:
       - import 'server-only'
       - import { cache } from 'react'
       - import { cookies } from 'next/headers'
       - import { redirect } from 'next/navigation'
       - import { decrypt } from './session'
       - export const verifySession = cache(async () => {
           const cookie = (await cookies()).get('session')?.value
           const session = await decrypt(cookie)
           if (!session?.userId) redirect('/login')
           return { isAuth: true, userId: session.userId as string, role: session.role as string, email: session.email as string, name: session.name as string, accessToken: session.accessToken as string }
         })

    7. Create `src/lib/api.ts`:
       - Helper for authenticated backend API calls from server components/actions
       - export async function fetchAPI(path: string, options?: RequestInit): fetches `${process.env.BACKEND_URL}${path}` with { ...options, headers: { 'Content-Type': 'application/json', ...options?.headers } }
       - export async function fetchAuthAPI(path: string, options?: RequestInit): calls verifySession(), adds Authorization: Bearer {session.accessToken} header, calls fetchAPI
       - Both handle error responses (non-ok status) by throwing or returning error objects

    8. Create `src/middleware.ts`:
       - import { NextRequest, NextResponse } from 'next/server'
       - import { decrypt } from './lib/session'
       - Protect routes: /tickets and /tickets/*
       - Read 'session' cookie, decrypt it
       - If no valid session, redirect to /login
       - If valid session and path is /login, redirect to /tickets
       - export const config = { matcher: ['/tickets/:path*', '/login'] }
       - IMPORTANT: middleware runs on Edge — jose works on Edge, this is fine

    9. Create `src/app/login/actions.ts`:
       - 'use server'
       - import { redirect } from 'next/navigation'
       - import { createSession } from '@/lib/session'
       - export async function login(prevState: any, formData: FormData):
         - Extract email, password from formData
         - Validate with zod (email string, password min 1 char)
         - If validation fails, return { error: 'Credenciais invalidas' }
         - POST to ${BACKEND_URL}/api/auth/login with { email, password }
         - If response not ok, return { error: 'Email ou senha incorretos' }
         - Parse response: { access_token, user }
         - await createSession(access_token, user)
         - redirect('/tickets')

    10. Create `src/app/login/page.tsx`:
        - Client component with useActionState (React 19) for the login server action
        - Form with email input, password input, submit button
        - Show error message from action state if present
        - Use shadcn Card, Input, Label, Button components
        - Centered layout, BKO Agent branding
        - Portuguese labels: "Email", "Senha", "Entrar"

    11. Update `src/app/layout.tsx`:
        - Import globals.css from correct new path
        - Add Inter font from next/font/google
        - Set metadata: title "BKO Agent", description "Plataforma de Tratamento Regulatorio Anatel"
        - Set html lang="pt-BR"

    IMPORTANT: Next.js 16 with React 19 uses `useActionState` (not `useFormState` which is deprecated). Import from 'react', not 'react-dom'.
    IMPORTANT: In server actions, `redirect()` throws internally (NEXT_REDIRECT). Do NOT wrap it in try/catch. Call redirect AFTER createSession, outside any try block.
    IMPORTANT: The session cookie stores the backend JWT encrypted. When calling backend APIs, extract the accessToken from the session and send it as Bearer header.
