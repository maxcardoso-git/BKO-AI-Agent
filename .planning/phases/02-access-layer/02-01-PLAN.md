    1. Install auth dependencies:

       `npm install -D @types/passport-local @types/passport-jwt @types/bcrypt`

    2. Create User entity at `backend/src/modules/operacao/entities/user.entity.ts`:
       - Fields: id (UUID PK), email (varchar, unique), passwordHash (varchar), name (varchar), role (varchar, default 'operator'), isActive (boolean, default true), createdAt, updatedAt
       - Use enum `UserRole { OPERATOR = 'operator', SUPERVISOR = 'supervisor', ADMIN = 'admin' }`
       - Place in operacao module because users operate on complaints
       - Add User to OperacaoModule's TypeOrmModule.forFeature array

    3. Create migration `1773774006000-CreateUserTable.ts`:
       - Raw SQL via queryRunner.query (follow project pattern from existing migrations)
       - CREATE TABLE "user" (id UUID DEFAULT uuid_generate_v4() PRIMARY KEY, email VARCHAR NOT NULL UNIQUE, "passwordHash" VARCHAR NOT NULL, name VARCHAR NOT NULL, role VARCHAR NOT NULL DEFAULT 'operator', "isActive" BOOLEAN NOT NULL DEFAULT true, "createdAt" TIMESTAMP NOT NULL DEFAULT now(), "updatedAt" TIMESTAMP NOT NULL DEFAULT now())
       - CREATE UNIQUE INDEX idx_user_email ON "user"(email)
       - Down: DROP TABLE "user"

    4. Create auth module structure:

       `dto/login.dto.ts`: class LoginDto with @IsEmail() email and @IsNotEmpty() password, both validated with class-validator

       `decorators/roles.decorator.ts`: export ROLES_KEY = 'roles', export Roles = (...roles: UserRole[]) => SetMetadata(ROLES_KEY, roles)

       `decorators/public.decorator.ts`: export IS_PUBLIC_KEY = 'isPublic', export Public = () => SetMetadata(IS_PUBLIC_KEY, true)

       `strategies/local.strategy.ts`: PassportStrategy(Strategy, 'local'), validates email+password via authService.validateUser, throws UnauthorizedException on failure

       `strategies/jwt.strategy.ts`: PassportStrategy(Strategy, 'jwt'), extracts Bearer token from Authorization header, looks up user by payload.sub (userId) via userRepository, returns user object to req.user. Use jwtFromRequest: ExtractJwt.fromAuthHeaderAsBearerToken(). secretOrKey from configService.get('JWT_SECRET').

       `guards/jwt-auth.guard.ts`: extends AuthGuard('jwt'), override canActivate to check IS_PUBLIC_KEY metadata via Reflector — if @Public() is set, allow through without auth

       `guards/roles.guard.ts`: implements CanActivate, reads ROLES_KEY from Reflector, if no roles required returns true, otherwise checks req.user.role is in required roles. Return false (403) if role not matched.

       `auth.service.ts`:
       - Inject Repository<User> and JwtService
       - validateUser(email, password): find user by email, bcrypt.compare password against passwordHash, return user (without passwordHash) or null
       - login(user): sign JWT payload { sub: user.id, email: user.email, role: user.role } with jwtService.sign(), return { access_token, user: { id, email, name, role } }

       `auth.controller.ts`:
       - POST /api/auth/login: @Public() @UseGuards(AuthGuard('local')), receives req.user from LocalStrategy, calls authService.login(req.user)
       - GET /api/auth/me: returns req.user (protected by global JWT guard, no @Public())

       `auth.module.ts`:
       - imports: TypeOrmModule.forFeature([User]), PassportModule, JwtModule.registerAsync with ConfigService (secret from JWT_SECRET, expiresIn from JWT_EXPIRES_IN or default '8h')
       - providers: AuthService, LocalStrategy, JwtStrategy, { provide: APP_GUARD, useClass: JwtAuthGuard }, { provide: APP_GUARD, useClass: RolesGuard }
       - controllers: AuthController
       - exports: AuthService

    5. Update app.module.ts: import AuthModule

    6. Add to .env.example:
       JWT_SECRET=your-jwt-secret-change-in-production
       JWT_EXPIRES_IN=8h

    7. Add JWT_SECRET and JWT_EXPIRES_IN to ConfigModule Joi validation in app.module.ts:

       JWT_EXPIRES_IN: Joi.string().default('8h')

    8. Add JWT_SECRET and JWT_EXPIRES_IN to backend/.env file (same values as .env.example for dev)

    IMPORTANT: Do NOT use enum type in PostgreSQL — use VARCHAR for role column (project decision from 01-02).

    IMPORTANT: Mark the existing AppController /api/health endpoint with @Public() so it remains accessible without auth.
