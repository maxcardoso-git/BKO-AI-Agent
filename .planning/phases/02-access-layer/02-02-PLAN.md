  <action>
    1. Create `dto/complaint-filter.dto.ts`:
       - Query params class with all optional fields:
         - status?: string (validated against ComplaintStatus enum values)
         - tipologyId?: string (@IsUUID)
         - subtipologyId?: string (@IsUUID)
         - situationId?: string (@IsUUID)
         - riskLevel?: string (validated against ComplaintRiskLevel enum values)
         - isOverdue?: string ('true'/'false' — query params are strings, parse in service)
         - search?: string (free text search on protocolNumber or rawText)
         - page?: number (@IsInt @Min(1), default 1)
         - limit?: number (@IsInt @Min(1) @Max(100), default 20)
         - sortBy?: string (default 'createdAt')
         - sortOrder?: 'ASC' | 'DESC' (default 'DESC')
       - Use @IsOptional() on all fields
       - Use @Type(() => Number) from class-transformer for page/limit

    2. Create `dto/complaint-list-response.dto.ts`:
       - Interface/class with: data (Complaint[]), total (number), page (number), limit (number), totalPages (number)

    3. Create `services/complaint.service.ts`:
       - Inject Repository<Complaint>
       - findAll(filters: ComplaintFilterDto): Promise<ComplaintListResponse>
         - Use createQueryBuilder('complaint')
         - leftJoinAndSelect complaint.tipology, complaint.subtipology, complaint.situation, complaint.regulatoryAction (avoid N+1)
         - Apply WHERE clauses conditionally based on filter presence:
           - complaint.status = :status
           - complaint.tipologyId = :tipologyId
           - complaint.subtipologyId = :subtipologyId
           - complaint.situationId = :situationId
           - complaint.riskLevel = :riskLevel
           - complaint.isOverdue = :isOverdue (parse 'true'/'false' to boolean)
           - For search: (complaint.protocolNumber ILIKE :search OR complaint.rawText ILIKE :search) with %search%
         - orderBy(complaint.{sortBy}, sortOrder)
         - skip((page - 1) * limit).take(limit)
         - getManyAndCount() for total
         - Return { data, total, page, limit, totalPages: Math.ceil(total / limit) }

       - findOne(id: string): Promise<Complaint>
         - Use createQueryBuilder('complaint')
         - leftJoinAndSelect ALL relations: tipology, subtipology, situation, regulatoryAction, details, history, attachments
         - where complaint.id = :id
         - getOneOrFail() — throws EntityNotFoundError which NestJS maps to 404
         - Order history by createdAt DESC, details by fieldName ASC

    4. Create `controllers/complaint.controller.ts`:
       - @Controller('complaints')
       - GET / — @Query() filters: ComplaintFilterDto, returns complaintService.findAll(filters)
       - GET /:id — @Param('id', ParseUUIDPipe) id, returns complaintService.findOne(id)
       - Both endpoints are automatically protected by global JWT guard (no @Public)
       - Add @UsePipes(new ValidationPipe({ transform: true, whitelist: true })) at controller level

    5. Update operacao.module.ts:
       - Add ComplaintService to providers
       - Add ComplaintController to controllers
       - Keep existing TypeOrmModule.forFeature and exports

    IMPORTANT: Use leftJoinAndSelect (not leftJoin) to avoid N+1 queries on the list endpoint.
    IMPORTANT: Do NOT add a global ValidationPipe in main.ts if one doesn't exist yet — add it. Check main.ts first. If it already has one, skip. The controller-level pipe is a fallback.
    IMPORTANT: Add a global ValidationPipe in main.ts with transform: true and whitelist: true if not already present — this ensures all DTOs are validated and transformed.

    1. Create `services/execution.service.ts`:
       - Inject Repository<TicketExecution>, Repository<Artifact>, Repository<AuditLog>

       - findByComplaintId(complaintId: string): Promise<TicketExecution[]>
         - createQueryBuilder('execution')
         - where execution.complaintId = :complaintId
         - leftJoinAndSelect execution.stepExecutions (load step details)
         - leftJoinAndSelect stepExecution.stepDefinition (show which step)
         - orderBy execution.createdAt DESC
         - getMany()

       - findArtifactsByComplaintId(complaintId: string): Promise<Artifact[]>
         - createQueryBuilder('artifact')
         - innerJoin artifact.stepExecution (need stepExecution to reach ticketExecution)
         - innerJoin stepExecution.ticketExecution
         - where ticketExecution.complaintId = :complaintId
         - leftJoinAndSelect artifact.stepExecution (include step context)
         - orderBy artifact.createdAt DESC
         - getMany()

       - findAuditLogsByComplaintId(complaintId: string): Promise<AuditLog[]>
         - where entityType = 'complaint' AND entityId = :complaintId
         - orderBy createdAt DESC
         - find() with simple repository query

    2. Create `controllers/execution.controller.ts`:
       - @Controller('complaints/:complaintId')
       - GET /executions — findByComplaintId(complaintId)
       - GET /artifacts — findArtifactsByComplaintId(complaintId)
       - GET /logs — findAuditLogsByComplaintId(complaintId)
       - All params validated with ParseUUIDPipe
       - All automatically protected by global JWT guard
