<task type="auto">
  <name>Task 1: Scaffold NestJS backend with TypeORM, pgvector, and Docker Compose</name>
  <files>
    docker-compose.yml
    backend/ (entire NestJS project via nest new)
    backend/src/database/data-source.ts
    backend/src/database/init.sql
    backend/src/database/migrations/000-EnableExtensions.ts
    backend/src/main.ts
    backend/src/app.module.ts
    backend/.env.example
    backend/.env
    backend/Dockerfile
    backend/typeorm.config.ts
  </files>
  <action>
    1. Run `nest new backend --package-manager npm --skip-git` from the project root to scaffold the NestJS project.

    2. Install backend dependencies:
       ```
       cd backend && npm install --save @nestjs/typeorm typeorm pg pgvector @nestjs/config class-validator class-transformer joi typeorm-extension @faker-js/faker
       ```

    3. Create `backend/src/database/data-source.ts` following the RESEARCH.md Pattern 1 exactly:
       - DataSourceOptions with type postgres, env vars for host/port/user/pass/db
       - entities: ['dist/**/*.entity.js']
       - migrations: ['dist/database/migrations/*.js']
       - seeds: ['dist/database/seeds/*.js']
       - synchronize: false (CRITICAL — never true, see RESEARCH pitfall 1)
       - Import dotenv and call dotenv.config()

    4. Update `backend/src/main.ts`:
       - Import pgvector from 'pgvector/typeorm' and call pgvector.registerTypes() BEFORE NestFactory.create() (see RESEARCH pitfall 4)
       - Set global prefix 'api'
       - Listen on port 3001

    5. Update `backend/src/app.module.ts`:
       - ConfigModule.forRoot with isGlobal: true and Joi validation schema for DB_HOST, DB_PORT (default 5432), DB_USER, DB_PASS, DB_NAME
       - TypeOrmModule.forRootAsync with useFactory injecting ConfigService
       - autoLoadEntities: true, synchronize: false, migrationsRun: true

    6. Create `backend/src/database/init.sql`:
       ```sql
       CREATE EXTENSION IF NOT EXISTS vector;
       CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
       ```

    7. Create migration `backend/src/database/migrations/000-EnableExtensions.ts` as shown in RESEARCH.md code examples (EnableExtensions000 class with CREATE EXTENSION IF NOT EXISTS vector and uuid-ossp).

    8. Create `backend/.env.example` with all required vars:
       ```
       DB_HOST=localhost
       DB_PORT=5432
       DB_USER=bko
       DB_PASS=bko_secret
       DB_NAME=bkoagent
       EMBEDDING_DIMENSIONS=1536
       NODE_ENV=development
       ```
       Copy to `backend/.env` (add .env to .gitignore).

    9. Add migration/seed scripts to `backend/package.json`:
       ```json
       "migration:generate": "npm run build && typeorm migration:generate -d dist/database/data-source.js src/database/migrations/$NAME",
       "migration:run": "npm run build && typeorm migration:run -d dist/database/data-source.js",
       "migration:revert": "npm run build && typeorm migration:revert -d dist/database/data-source.js",
       "seed": "npm run build && node dist/database/seeds/run.js"
       ```

    10. Create `backend/Dockerfile`:
        - FROM node:20-alpine
        - WORKDIR /app
        - COPY package*.json, npm install, COPY . ., npm run build
        - CMD ["node", "dist/main.js"]

    11. Create `docker-compose.yml` at project root following RESEARCH.md Pattern 5:
        - postgres service: image pgvector/pgvector:pg17, env vars, port 5432, volume for pgdata AND init.sql mount to docker-entrypoint-initdb.d, healthcheck (pg_isready)
        - redis service: redis:7-alpine, port 6379
        - backend service: build ./backend, depends_on postgres (service_healthy), env vars for DB, port 3001
        - frontend service: build ./frontend, port 3000
        - volumes: pgdata

    12. Create a root `.gitignore` with: node_modules, dist, .env, *.env.local, .DS_Store

    13. Create `backend/.gitignore` adding: dist/, node_modules/, .env
  </action>
  <verify>
    - `cd backend && npm run build` completes without errors
    - `docker compose up postgres redis -d` starts PostgreSQL and Redis
    - `docker compose exec postgres psql -U bko -d bkoagent -c "SELECT 1"` returns 1
    - `docker compose exec postgres psql -U bko -d bkoagent -c "CREATE EXTENSION IF NOT EXISTS vector; SELECT vector '[1,2,3]';"` succeeds
  </verify>
  <done>
    NestJS backend project exists with TypeORM configured (synchronize:false), pgvector registered in main.ts, Docker Compose starts PostgreSQL with pgvector extension available, and Redis is running.
  </done>
