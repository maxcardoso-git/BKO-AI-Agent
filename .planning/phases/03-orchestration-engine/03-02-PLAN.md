**Define ExecutionContext interface at the top of the file (not exported — internal only):**
```typescript
interface ExecutionContext {
  tipologyKey: string;
  situationKey: string | null;
  slaBusinessDays: number;
  slaDeadline: string;
  selectedActionKey: string | null;
  stepOutputs: Record<string, Record<string, unknown>>;
}
```

**Method 1: `async startExecution(complaintId: string): Promise<TicketExecution>`**

1. Load complaint with relations: `await this.complaintRepo.findOne({ where: { id: complaintId }, relations: ['tipology', 'situation'] });`
2. If not found: throw HttpException('Complaint not found', 404).
3. If `!complaint.tipologyId`: throw HttpException('Complaint has no tipology assigned', 422).
4. Concurrent guard: `await this.ticketExecutionRepo.findOne({ where: { complaintId, status: In([TicketExecutionStatus.RUNNING, TicketExecutionStatus.PAUSED_HUMAN]) } });` If found: throw HttpException('An active execution already exists for this complaint', 409).
5. Select capability: `const capabilityVersion = await this.orchService.selectCapabilityVersion(complaint.tipologyId);`
6. Load steps sorted: `const steps = await this.stepDefinitionRepo.find({ where: { capabilityVersionId: capabilityVersion.id, isActive: true }, order: { stepOrder: 'ASC' } });`
   PITFALL: Always sort by stepOrder explicitly. TypeORM relation loading does NOT guarantee order.
7. If steps.length === 0: throw HttpException('Capability version has no active steps', 422).
8. Compute SLA: `const slaResult = this.orchService.computeSla(complaint.createdAt, complaint.tipology!, complaint.situation ?? null);`
9. Update complaint: `complaint.slaDeadline = slaResult.slaDeadline; complaint.slaBusinessDays = slaResult.slaBusinessDays; complaint.isOverdue = slaResult.isOverdue; complaint.status = ComplaintStatus.IN_PROGRESS; await this.complaintRepo.save(complaint);`
10. Build initial metadata:
    ```typescript
    const metadata: ExecutionContext = {
      tipologyKey: complaint.tipology!.key,
      situationKey: complaint.situation?.key ?? null,
      slaBusinessDays: slaResult.slaBusinessDays,
      slaDeadline: slaResult.slaDeadline.toISOString(),
      selectedActionKey: null,
      stepOutputs: {},
    };
    ```
11. Create execution: `this.ticketExecutionRepo.create({ complaintId, capabilityVersionId: capabilityVersion.id, status: TicketExecutionStatus.RUNNING, startedAt: new Date(), currentStepKey: steps[0].key, metadata });`
12. Save and return: `return this.ticketExecutionRepo.save(execution);`
13. Write audit log: `await this.auditLogRepo.save(this.auditLogRepo.create({ action: 'execution_started', entityType: 'ticket_execution', entityId: execution.id, details: { complaintId, capabilityVersionId: capabilityVersion.id } }));`
14. Return the execution (after save, before audit log is fine — audit is fire-and-forget in Phase 3).

**Method 2: `async advanceStep(executionId: string, operatorInput?: Record<string, unknown>): Promise<StepExecution>`**

1. Load execution: `await this.ticketExecutionRepo.findOne({ where: { id: executionId } });`
2. If not found: throw HttpException('Execution not found', 404).
3. Guard status: if execution.status is not RUNNING and not PAUSED_HUMAN: throw HttpException('Execution is not in a state that can advance. Current status: ' + execution.status, 422).
4. Load complaint with relations: `await this.complaintRepo.findOne({ where: { id: execution.complaintId }, relations: ['tipology', 'situation'] });`
5. Load all steps for the capability version, sorted by stepOrder ASC.
6. Find current step: `const currentStep = steps.find(s => s.key === execution.currentStepKey);` If not found: throw HttpException('Current step key not found: ' + execution.currentStepKey, 422).
7. Run policy validator: `const policyResult = await this.orchService.validatePolicyRules(complaint!, 'advance');` If !policyResult.passed: throw HttpException({ message: 'Step blocked by policy', violations: policyResult.violations }, 422).
8. If `currentStep.isHumanRequired && !operatorInput`:
   - Create/update step execution: `const stepExec = this.stepExecutionRepo.create({ ticketExecutionId: executionId, stepDefinitionId: currentStep.id, stepKey: currentStep.key, status: StepExecutionStatus.WAITING_HUMAN, startedAt: new Date(), input: (execution.metadata as ExecutionContext) });`
   - Save stepExec.
   - Update execution: `execution.status = TicketExecutionStatus.PAUSED_HUMAN; await this.ticketExecutionRepo.save(execution);`
   - Update complaint: `complaint!.status = ComplaintStatus.WAITING_HUMAN; await this.complaintRepo.save(complaint!);`
   - Return stepExec.
9. If execution.status === TicketExecutionStatus.PAUSED_HUMAN (operator providing input to resume):
   - Set execution.status = TicketExecutionStatus.RUNNING.
10. Execute the skill stub:
    - Find skill binding: `const binding = await this.stepSkillBindingRepo.findOne({ where: { stepDefinitionId: currentStep.id, isActive: true }, relations: ['skillDefinition'] });`
    - Build input: the execution metadata (context) merged with operatorInput if provided.
    - Execute: `const output = binding ? this.executeSkillStub(binding.skillDefinition.key, skillInput) : { result: 'no_skill_bound' };`
    - Track timing: record startTime before, compute durationMs after.
11. Create StepExecution record: `{ ticketExecutionId: executionId, stepDefinitionId: currentStep.id, stepKey: currentStep.key, status: StepExecutionStatus.COMPLETED, startedAt, completedAt: new Date(), durationMs, input: skillInput, output, retryCount: 0 }`
12. Save stepExec.
13. Accumulate output: `const meta = (execution.metadata ?? {}) as ExecutionContext; meta.stepOutputs[currentStep.key] = output as Record<string, unknown>;`
    If skill is `DetermineRegulatoryAction` and output has `actionKey`: set `meta.selectedActionKey = output.actionKey`.
14. Determine next step: find the step with `stepOrder === currentStep.stepOrder + 1` in the sorted steps array (or simply the next in the sorted array after currentStep). If no next step exists, this is the last step.
15. Update execution: `execution.currentStepKey = nextStep?.key ?? null; execution.metadata = meta;` If no next step: `execution.status = TicketExecutionStatus.COMPLETED; execution.completedAt = new Date(); execution.totalDurationMs = new Date().getTime() - execution.startedAt!.getTime();`
16. Save execution.
17. If execution completed, update complaint: `complaint!.status = ComplaintStatus.COMPLETED; await this.complaintRepo.save(complaint!);`
18. Write audit log with action='step_completed', entityType='step_execution', entityId=stepExec.id, details: { stepKey, executionId, status }.
19. Return stepExec.

**Method 3: `async finalizeExecution(executionId: string, outcome?: Record<string, unknown>): Promise<TicketExecution>`**

1. Load execution. Guard: status must be RUNNING or PAUSED_HUMAN.
2. Load complaint with tipology relation.
3. Run policy validator with action='finalizar': `await this.orchService.validatePolicyRules(complaint!, 'finalizar');` If blocked, throw 422 with violations.
4. Update execution: `status = COMPLETED, completedAt = new Date(), totalDurationMs = now - startedAt, metadata.outcome = outcome (if provided)`.
5. Save execution.
6. Update complaint: `status = ComplaintStatus.COMPLETED`.
7. Write audit log: action='execution_finalized'.
8. Return execution.

**Method 4: `async retryStep(executionId: string, stepKey: string): Promise<StepExecution>`**

1. Load execution. If not found: 404.
2. Find StepExecution: `await this.stepExecutionRepo.findOne({ where: { ticketExecutionId: executionId, stepKey } });` If not found: throw HttpException('No step execution found for key: ' + stepKey, 404).
3. If stepExec.status is not FAILED and not WAITING_HUMAN: throw HttpException('Step is not in a retryable state: ' + stepExec.status, 422).
4. Find the skill binding for the step and re-execute the skill stub.
5. Update stepExec: `retryCount += 1, status = COMPLETED, startedAt = new Date(), completedAt = new Date(), durationMs = computed, output = newOutput, errorMessage = null`.
6. Save stepExec.
7. If execution.status === TicketExecutionStatus.FAILED: `execution.status = TicketExecutionStatus.RUNNING; await this.ticketExecutionRepo.save(execution);`
8. Update metadata.stepOutputs[stepKey] with new output.
9. Write audit log: action='step_retried'.
10. Return stepExec.
