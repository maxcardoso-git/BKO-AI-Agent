**Method 1: `computeSla(createdAt: Date, tipology: Tipology, situation: Situation | null): { slaDeadline: Date; slaBusinessDays: number; isOverdue: boolean }`**

Logic:
1. `const slaBusinessDays = situation?.slaOverrideDays ?? tipology.slaBusinessDays;`
2. Compute slaDeadline using `addBusinessDays(createdAt, slaBusinessDays)` — a private helper method.
3. `addBusinessDays` loops day-by-day: skip Saturday (day 6) and Sunday (day 0). No holiday handling in Phase 3.
4. `isOverdue = new Date() > slaDeadline`
5. Return `{ slaDeadline, slaBusinessDays, isOverdue }`

This is a synchronous method (no async needed — pure computation on already-loaded entities).

**Method 2: `async selectCapabilityVersion(tipologyId: string): Promise<CapabilityVersion>`**

Logic:
1. `const capability = await this.capabilityRepo.findOne({ where: { tipologyId, isActive: true }, relations: ['versions'] });`
2. If no capability: `throw new HttpException('No active capability found for tipologyId: ' + tipologyId, 422);`
3. `const currentVersion = capability.versions.find(v => v.isCurrent && v.isActive);`
4. If no currentVersion: `throw new HttpException('No current active version for capability: ' + capability.key, 422);`
5. Return currentVersion.

PITFALL: Capability.tipologyId is nullable. The query `{ tipologyId, isActive: true }` handles this correctly — TypeORM matches the exact value.

**Method 3: `async validatePolicyRules(complaint: Complaint, action: 'advance' | 'finalizar'): Promise<{ passed: boolean; violations: string[] }>`**

Logic:
1. Query all BLOCKING + active rules: `await this.ruleRepo.find({ where: { ruleType: RegulatoryRuleType.BLOCKING, isActive: true } });`
2. Filter applicable rules:
   - If rule.tipologyId is set AND does not match complaint.tipologyId, skip.
   - If rule.metadata has `blocks_action` AND it does not match the `action` param, skip.
3. For each applicable rule, evaluate:
   - For `requires_complete_checklist` metadata: In Phase 3, this is a stub — always passes. Add a comment: `// Phase 4: check actual checklist completion`.
   - For any other rule: add rule.title to violations array (conservative — block if we don't know how to evaluate).
4. Return `{ passed: violations.length === 0, violations }`.

Import `HttpException` from `@nestjs/common`. Import `RegulatoryRuleType` from the regulatory-rule entity. Import `Complaint` type from operacao entities (type-only import for the parameter signature).

Do NOT import `In` from TypeORM in this file — it's not needed here. The `In` operator is needed in Plan 03-02's TicketExecutionService.
